# Αλγόριθμοι για Δεδομένα Ευρείας Κλίμακας — Project 1

Πρώτη εργασία στο μάθημα **Αλγόριθμοι για Δεδομένα Ευρείας Κλίμακας**, Τμήμα Μηχανικών Η/Υ & Πληροφορικής, Πανεπιστήμιο Ιωαννίνων.

Το project υλοποιεί και αξιολογεί πειραματικά μια σειρά από **streaming / probabilistic αλγορίθμους**: αλγορίθμους που επεξεργάζονται τεράστιες ροές δεδομένων χρησιμοποιώντας ελάχιστη μνήμη (συνήθως υπογραμμική ως προς το πλήθος των στοιχείων), δίνοντας προσεγγιστικές αλλά θεωρητικά τεκμηριωμένες απαντήσεις. Κάθε ενότητα συνοδεύεται από πειράματα και διαγράμματα (`plots/`) που επαληθεύουν εμπειρικά τις θεωρητικές εγγυήσεις.

## Αναλυτική περιγραφή ενοτήτων

### Ενότητα A — Morris Counter (probabilistic counting)

Ο **Morris counter** μετράει `n` γεγονότα χρησιμοποιώντας μόνο `O(log log n)` bits αντί για τα `O(log n)` που θα απαιτούσε ένας ακριβής μετρητής. Αντί να αυξάνει τον μετρητή `C` σε κάθε γεγονός, τον αυξάνει **πιθανοτικά** με πιθανότητα `1/α^C`, και επιστρέφει την εκτίμηση `(α^C − 1)/(α − 1)`.

- **`A1_morris_counter.py`** — Υλοποίηση του βασικού μετρητή. Πειράματα για `n = 1.000.000` με διάφορες τιμές του `α`, μελέτη της διασποράς, βελτίωση της ακρίβειας με **mean/median πολλαπλών αντιγράφων** (averaging για μείωση variance, median για robustness), εκδοχή περιορισμένη σε **8 bits**, και επαλήθευση Monte-Carlo.
- **`A2_morris_exact.py`** — **Ακριβής** (όχι Monte-Carlo) υπολογισμός της κατανομής του `C` βήμα-βήμα με `Fraction` (ρητοί αριθμοί, χωρίς σφάλμα στρογγυλοποίησης). Υπολογίζει αναλυτικά την πιθανότητα η εκτίμηση να πέφτει στο διάστημα `[n/c, c·n]` και αξιολογεί την επίδραση του αριθμού των bits.

### Ενότητα B — Μέτρηση μοναδικών στοιχείων (distinct elements)

Το πρόβλημα: πόσα **διακριτά** στοιχεία περιέχει μια ροή, με μία μόνο διέλευση και υπογραμμική μνήμη.

- **`B1_distinct_counting.py`** — Εκτίμηση βασισμένη στα **trailing zeros**: ο αριθμός των μηδενικών στο τέλος των hashed τιμών συνδέεται με το πλήθος των μοναδικών στοιχείων (αν έχουμε δει `d` μοναδικά, αναμένουμε max trailing zeros `≈ log₂ d`). Περιλαμβάνει υλοποίηση **Trie** (δυαδικό δέντρο προθεμάτων) για ακριβή μέτρηση αναφοράς, και πειράματα σε ομοιόμορφες και μη-ομοιόμορφες κατανομές.
- **`B2_hash_family.py`** — Υλοποίηση **universal hash family** της μορφής `h(x) = ((αx + β) mod p) mod 2^d` με `p` πρώτο. Εμπειρική επαλήθευση των ιδιοτήτων που πρέπει να ικανοποιεί μια καλή οικογένεια hash (π.χ. `P(tr(h(x)) ≥ r) ≈ 1/2^r`, ανεξαρτησία, ομοιόμορφη κατανομή).
- **`B3_bjkst.py`** — Ο αλγόριθμος **BJKST** (Bar-Yossef–Jayram–Kumar–Sivakumar–Trevisan), ένας πιο εξελιγμένος (ε, δ)-εκτιμητής μοναδικών στοιχείων. Διατηρεί τα `k` μικρότερα hash values (μέσω max-heap) και εκτιμά το πλήθος. Πειράματα για διαφορετικά `ε` (`0.05`, `0.1`) σε ομοιόμορφα και μη-ομοιόμορφα δεδομένα, με μελέτη του trade-off ακρίβειας / μνήμης.

### Ενότητα C — Bloom Filters

- **`C1_bloom_intersection.py`** — Υλοποίηση **Bloom filter** (πίνακας `m` bits + `k` hash functions) για προσεγγιστικό έλεγχο συμμετοχής (membership) χωρίς false negatives, με ελεγχόμενο ποσοστό **false positives**. Εφαρμογή στον υπολογισμό της **τομής** δύο συνόλων από 100-bit strings, και μελέτη του πώς οι παράμετροι `m`, `k` επηρεάζουν το σφάλμα.

### Ενότητα D — Packet Tracing

- **`D_packet_tracing.py`** — Εφαρμογή των Bloom filters σε σενάριο **δικτύου**: ανίχνευση της διαδρομής πακέτων (packet tracing) μέσα σε γράφο δρομολογητών. Κάθε router καταγράφει τα πακέτα που διέρχονται σε ένα Bloom filter· από τα filters ανακατασκευάζεται η διαδρομή. Πείραμα με `100.000` μηνύματα και `10` source routers, ανάλυση false positives στην ανακατασκευή.

## Εκτέλεση

Απαιτείται Python 3 με `matplotlib`:

```bash
pip install matplotlib
```

Εκτέλεση όλων των πειραμάτων (παράγει όλα τα διαγράμματα στο `plots/`):

```bash
cd solutions_4940
python run_all.py
```

Ή μεμονωμένα μια ενότητα, π.χ.:

```bash
cd solutions_4940
python B3_bjkst.py
```

> Σημείωση: όλα τα scripts χρησιμοποιούν `random.seed(42)` ώστε τα αποτελέσματα και τα διαγράμματα να είναι **αναπαραγώγιμα**.
